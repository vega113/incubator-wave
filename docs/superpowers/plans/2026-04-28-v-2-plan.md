# V-2 — Strip developer strings from J2CL product UI

Issue: [#1100](https://github.com/vega113/supawave/issues/1100)
Umbrella: [#1098](https://github.com/vega113/supawave/issues/1098)
Parent: [#904](https://github.com/vega113/supawave/issues/904)
Roadmap: `docs/superpowers/specs/2026-04-28-j2cl-visual-polish-roadmap.md`
Mockup target (functional, no V-2-specific mock — V-2 simply removes chrome from the V-1 layout): `docs/superpowers/mockups/2026-04-28-j2cl-functional-ui/01-shell-inbox-with-waves.svg`

## 1. Goal

`?view=j2cl-root` for a default signed-in user must not render any of:

- `Read.` — emitted by `J2clSelectedWaveModel.formatUnreadText` into `.sidecar-selected-unread`.
- `Live updates connected.` / `Live updates reconnected.` — emitted by `J2clSelectedWaveProjector.buildStatusText` into `.sidecar-selected-status`.
- The channel/snapshot URL line `wavelet · channel chN · snapshot vNNN` — emitted by `J2clSelectedWaveProjector.buildDetailText` into `.sidecar-selected-detail`.
- `Opened wave` — emitted by `J2clSelectedWaveView.mountColdCard` and by `HtmlRenderer.appendJ2clRootShellSelectedSnapshot` (and the preview-fixture renderer) into `.sidecar-eyebrow`.
- `Reply target: b+<id>` — emitted by `wavy-composer.js` (`<p class="target">`) and `composer-inline-reply.js`.

These data points stay accessible to developers under a new experimental flag `j2cl-debug-overlay` (default off in prod). With the flag on, every string above renders again in the same DOM location (no extra overlay UI required for this slice — the existing elements simply un-hide).

The J-UI version-history "Recent / Next unread / Previous / Next / End / Archive / Unpin" tab strip mentioned in the V-2 roadmap entry is a redesign owned by V-1 (rebuilt nav row); V-2 only suppresses dev strings.

## 2. Approach

A single hide mechanism scoped to the document body:

1. Add `j2cl-debug-overlay` to `KnownFeatureFlags` (default off, default-empty allowlist).
2. `WaveClientServlet` resolves the flag per viewer and forwards it to a new `renderJ2clRootShellPage` overload.
3. `HtmlRenderer.renderJ2clRootShellPage` adds CSS class `j2cl-debug-overlay-on` to `<body>` when the flag is on. The body already carries `j2cl-root-shell-page`, so the new class composes cleanly.
4. A new CSS rule lives next to the J2CL stylesheet:
   ```
   body.j2cl-root-shell-page:not(.j2cl-debug-overlay-on) [data-j2cl-debug-only] { display: none !important; }
   ```
   We add this to `j2cl/src/main/webapp/styles/sidecar.css` (the file already loaded by the J2CL shell). A single rule covers both the SSR DOM and any post-upgrade Lit/plain-DOM elements that mark themselves as debug-only.
5. Mark every dev-string element with `data-j2cl-debug-only="true"`:
   - `HtmlRenderer.appendJ2clRootShellSelectedSnapshot`: eyebrow, status, detail.
   - `HtmlRenderer.appendJ2clRootShellPreviewFixtureSelectedCard` (the `Read-surface preview fixture` block at line ~3866): eyebrow.
   - `J2clSelectedWaveView.mountColdCard`: eyebrow, status, detail elements via `setAttribute`.
   - `J2clSelectedWaveView.applyState` (and the server-first projector path at the same view): set the attribute on `unread` only when the rendered text is `Read.` (or the digest "Selected digest is read." variant). When the text becomes "X unread" the attribute is removed so the count is visible.
   - `wavy-composer.js` `<p class="target">`: add `data-j2cl-debug-only="true"`.
   - `composer-inline-reply.js` `<p class="target">`: add `data-j2cl-debug-only="true"`.

This keeps the model unchanged (`getUnreadText`, `getStatusText`, `buildDetailText` still return the same strings — many tests and controller paths consume them), and confines the visual change to "is this DOM node visible".

## 3. Files

### Wave server (Jakarta overrides)
- `wave/src/main/java/org/waveprotocol/box/server/persistence/KnownFeatureFlags.java` — register `j2cl-debug-overlay`.
- `wave/src/jakarta-overrides/java/org/waveprotocol/box/server/rpc/WaveClientServlet.java` — read flag, pass to renderer.
- `wave/src/jakarta-overrides/java/org/waveprotocol/box/server/rpc/HtmlRenderer.java` — add `debugOverlayEnabled` parameter (new overload, keep prior overloads as bridges defaulting to `false`); emit body class when on; tag eyebrow/status/detail with `data-j2cl-debug-only`.
- `j2cl/src/main/webapp/styles/sidecar.css` (or whichever sidecar.css is bundled — confirm during implement; likely `wave/web/static/...` or generated under `j2cl/src/main/webapp`) — add the `display: none` rule.

### J2CL client
- `j2cl/src/main/java/org/waveprotocol/box/j2cl/search/J2clSelectedWaveView.java` — set `data-j2cl-debug-only` attribute on eyebrow/status/detail when creating elements; toggle on `unread` based on read state.

### Lit elements
- `j2cl/lit/src/elements/wavy-composer.js` — add `data-j2cl-debug-only` to the reply-target paragraph.
- `j2cl/lit/src/elements/composer-inline-reply.js` — add `data-j2cl-debug-only` to the reply-target paragraph.

### Tests (update / add)
- `wave/src/test/java/org/waveprotocol/box/server/rpc/HtmlRendererJ2clRootShellIntegrationTest.java` — assert that with flag off the body lacks `j2cl-debug-overlay-on`, debug-only attrs are present on eyebrow/status/detail, and a new test asserts flag-on path adds the body class.
- `j2cl/src/test/java/org/waveprotocol/box/j2cl/search/J2clSelectedWaveViewChromeTest.java` — assert `data-j2cl-debug-only` is set on the cold-card eyebrow/status/detail and toggled on `unread`.
- `j2cl/lit/test/wavy-composer.test.js` (or the closest reply-target test) — assert reply-target paragraph carries the debug-only attribute.
- `j2cl/lit/test/composer-inline-reply.test.js` — same.

We do **not** change tests that assert `getUnreadText() == "Read."` (model contract is preserved). The visual change is DOM-only.

## 4. Feature-flag plumbing

Bridge overloads on `renderJ2clRootShellPage` keep callers compiling. Pattern matches the existing J-UI-1, J-UI-5, J-UI-8 flag plumbing. New overload signature appended at the end of the existing 12-arg form:

```java
public static String renderJ2clRootShellPage(
    JSONObject sessionJson, String analyticsAccount,
    String buildCommit, long serverBuildTime, String currentReleaseId,
    String rootShellReturnTarget, String websocketAddress,
    J2clSelectedWaveSnapshotRenderer.SnapshotResult snapshotResult,
    boolean railCardsEnabled,
    String viewerLocale,
    boolean serverFirstPaintEnabled,
    boolean inlineRichComposerEnabled,
    boolean debugOverlayEnabled);
```

Existing overloads delegate with `debugOverlayEnabled=false`.

## 5. Test strategy

- `sbt -Dwave.module=wave compile` (Jakarta sources) and `sbt test` for the J2CL search package and HtmlRenderer test class.
- Lit tests: `cd j2cl/lit && npm test -- --grep "composer"` (or full suite).
- Local server QA per §6.

## 6. Local-server QA (gate)

1. From this worktree: `cd wave && sbt compile` then start dev server on a free port (default 9900): `bash scripts/worktree-boot.sh --port 9900`.
2. Register a fresh user via the sign-up flow (do not assume `vega` exists).
3. Sign in and navigate to `?view=j2cl-root`. Open a wave so the selected-wave card is mounted.
4. Capture **before** screenshot from `origin/main` HEAD: switch to a separate scratch worktree pinned to main on a different port; capture; save under `docs/superpowers/screenshots/v-2/before.png`.
5. Switch back to this worktree, restart server; capture **after** screenshot at the same window size and same wave selection; save `docs/superpowers/screenshots/v-2/after.png`.
6. Verify the after screenshot has zero occurrences of `Read.`, `Live updates connected.`, the `· channel chN · snapshot vNNN` line, `Opened wave`, and `Reply target: b+`.
7. Toggle the flag on (`scripts/feature-flag.sh --local enable j2cl-debug-overlay && scripts/feature-flag.sh --local set j2cl-debug-overlay "..." --global` per the flag CLI), reload, capture `docs/superpowers/screenshots/v-2/after-debug-on.png` showing the strings restored.
8. Edge cases:
   - Empty selected-wave state (no wave open) — eyebrow/status/detail must remain hidden.
   - Reply target paragraph: open reply composer; verify hidden when flag off, visible when on.
   - `?view=gwt` path is unaffected — capture quick screenshot to confirm no regressions there (no body class change because `WaveClientServlet` only adds the new flag plumbing on the J2CL branch).

## 7. Feature flag + changelog

- Flag entry: `j2cl-debug-overlay` registered in `KnownFeatureFlags` (default off, no allowlist).
- Changelog fragment: `wave/config/changelog.d/2026-04-28-issue-1100-v-2.json`, summary noting that dev-only strings on `?view=j2cl-root` are now hidden by default and gated behind `j2cl-debug-overlay`.

## 8. Rollback

Disabling the slice is two lines: remove the `<body>` class branch and the `data-j2cl-debug-only` rule. With the flag on globally, behavior matches `main` exactly.

## 9. Out of scope

- Building a richer debug overlay UI (drawer, toggle button). The slice's "debug overlay" is "the existing dev strings, un-hidden by flag". Roadmap notes the overlay is just the hidden surface.
- Removing the version-history tab strip (V-1 redesign owns it).
- Touching the `aria-label` "Read." on `wavy-search-rail-card` — that is screen-reader content, not visible product UI; the issue scopes to the "rendered as content" `Read.` in the wave header.
- V-1 layout, V-3 toolbar, V-4 blip chrome.
